# Freelancer Negotiation Environment

An OpenEnv-compatible reinforcement learning environment for training and evaluating agents on realistic freelancer-client negotiations.

## Problem Description

Freelance work negotiations are multi-objective decision problems. Agents must balance:

- profitability (avoid underpricing),
- conversion (close deals when sensible),
- efficiency (few turns),
- professionalism (relevant and non-repetitive communication),
- robustness against adversarial behavior (toxic or contradictory client demands).

This environment models that tension in a deterministic simulation suitable for RL, policy benchmarking, and evaluator development.

## Why This Matters

Negotiation quality directly affects revenue, delivery risk, and client retention in marketplaces and agency workflows. A strong policy can:

- improve margin discipline,
- reduce low-quality deal acceptance,
- standardize communication quality,
- handle toxic behaviors consistently,
- make offline evaluation and regression testing repeatable.

The environment is designed to approximate these production constraints while remaining reproducible enough for robust grading.

## Environment Design

The simulator is implemented as a deterministic, rule-based OpenEnv server.

Core state:

- `current_price`
- `client_budget`
- `ideal_price`
- `deadline`
- `revisions`
- `conversation_history`
- `client_type` (`cheap`, `normal`, `premium`, `toxic`)

Episode behavior:

1. `reset()` rotates through deterministic scenarios.
2. `step(action)` processes agent intent + message.
3. Client response is generated by policy rules tied to `client_type`.
4. Reward is returned at every step.
5. Episode ends on accepted/rejected deal or max-step timeout.

## Action Space

`FreelancerNegotiationAction`

- `message: str` — natural-language response from the freelancer agent.
- `action_type: Enum` — one of:
  - `negotiate`
  - `accept`
  - `reject`

## Observation Space

`FreelancerNegotiationObservation`

- `client_message: str` — latest client utterance.
- `negotiation_state`
  - `current_price: float`
  - `deadline: str`
  - `revisions: int`
- `conversation_history: list[str]`
- `done: bool`
- `reward: float` (step reward)
- `metadata: dict` (includes diagnostic info such as reward components)

## Tasks (Easy / Medium / Hard)

Tasks are defined in `tasks.py` and graded deterministically.

### Easy

- Scenario: simple, low-friction client.
- Goal: close fairly near target price with concise dialogue.

### Medium

- Scenario: conflicting requirements (low budget + tight deadline).
- Goal: make a disciplined decision under constraint mismatch, including correct rejection when needed.

### Hard

- Scenario: toxic client with unclear requirements and revision pressure.
- Goal: enforce boundaries, keep communication professional, and avoid low-quality deals.

## Reward System

The reward function is step-based and balanced for realism:

- `+10` if final agreed price is close to `ideal_price`
- `-10` if accepted deal is too cheap
- `-8` if client is lost unnecessarily
- `+5` for proper handling of toxic behavior (boundary-setting)
- `+3` for efficient negotiation (few steps)
- `-3` for irrelevant or repetitive messages

Additional shaping:

- small relevance bonus for high-quality turns,
- per-step cost to discourage rambling,
- score clamping for stability.

## Setup Instructions

### 1. Install Python Dependencies

From the `freelancer_negotiation_env` directory:

```bash
python -m pip install -U pip
python -m pip install .
```

### 2. Run the OpenEnv Server Locally

```bash
python -m uvicorn freelancer_negotiation_env.server.app:app --host 0.0.0.0 --port 8000
```

### 3. (Optional) Build Docker Image

```bash
docker build -t freelancer-negotiation-env:latest -f server/Dockerfile .
docker run --rm -p 8000:8000 freelancer-negotiation-env:latest
```

## How To Run Inference

The repository includes `inference.py`, which runs all three tasks with an OpenAI-compatible model endpoint and reports final score.

### Required Environment Variables

- `API_BASE_URL` — OpenAI-compatible endpoint
- `MODEL_NAME` — model identifier
- `HF_TOKEN` — API key/token used by the OpenAI client

Optional:

- `OPENENV_BASE_URL` (default: `http://localhost:8000`)

### Example

```bash
export API_BASE_URL="https://your-endpoint/v1"
export MODEL_NAME="your-model"
export HF_TOKEN="your-token"
export OPENENV_BASE_URL="http://localhost:8000"

python ../inference.py
```

Expected output includes per-task summaries and aggregate metrics:

- per-task steps/decision/reward/score
- `final_score=...`
- `total_reward=...`
